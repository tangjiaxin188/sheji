## 设计理念
服务器不存储具体聊天信息，而是仅仅存储加密过的用户信息，保障去中心化的聊天安全。所有信息传递都需要进行私钥验证，提高了安全性。信息传递基于http协议和json格式，保证了高可用和复用性，也可使用https协议提高安全性。

## 依赖库
本服务器使用到了mongoose作为网络和http库，cjson作为json处理库，crypot作为加密库。客户端使用raylib绘制UI界面，用已有的开源项目极大的简化了开发过程，提高开发效率

## 具体流程
本软件分为服务端和客户端，客户端先进行帐号注册，服务端将编号和客户端生成的公钥存储起来，客户端发送消息时，会给服务端发送带自身编号和签名，以及目标编号的连接请求，服务端接收后会生成一个专属于本次聊天的加密密钥，二者再进行加密通信。

## 客户端数据处理
客户端会将文本内容进行加密，其他文件内容会先经过压缩再加密，降低加密内容复杂度和传输带宽。

## 服务端内容存储
服务端不长期存储聊天数据，仅作为中转站转发数据，通过流式转发降低性能消耗。服务器通过sqlite存储用户的信息，包括用户编号和sha256,sha1,sha128分别计算得到的密码，以确保不会泄漏密码。

## 编译方法
本项目同时使用了cmake和zig两个构建器，方便不同平台进行编译，以及交叉编译，降低开发复杂度。
使用以下指令
'''cd build/ && cmake .. && make -j4'''
或者
'''zig build'''
可以直接运行编译后的二进制文件，也可以使用
'''zig build server'''
'''zig build client'''
如果使用zig编译的话。
