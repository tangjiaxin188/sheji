# Ed25519 签名认证架构设计

## 认证流程

```
┌─────────────┐                          ┌─────────────┐
│   客户端     │                          │    服务端    │
└──────┬──────┘                          └──────┬──────┘
       │                                        │
       │  1. 生成密钥对 (本地)                   │
       │     sk[64], pk[32]                     │
       │                                        │
       │  ───── AUTH_REQUEST + 公钥 ──────────► │
       │                                        │ 2. 生成挑战 challenge[32]
       │                                        │    存储：client_id → (公钥，挑战)
       │                                        │
       │  ◄ ──── AUTH_CHALLENGE + challenge ─── │
       │                                        │
       │  3. 签名：sig = sign(sk, challenge)    │
       │                                        │
       │  ───── AUTH_RESPONSE + sig ──────────► │
       │                                        │ 4. 验证：check(sig, pk, challenge)
       │                                        │    ✓ → 生成 chat_token
       │                                        │
       │  ◄ ──── AUTH_SUCCESS + chat_token ──── │
       │                                        │
       │  5. 后续请求携带 chat_token ──────────► │
       │                                        │ 6. 验证 token → 查找会话
```

## 核心设计

### 1. 无状态认证

- **私钥永不传输**：客户端本地生成并保存，服务端只存储公钥
- **挑战 - 响应机制**：每次登录挑战数据随机，防止重放攻击
- **Token 作为会话凭证**：认证成功后生成，后续请求凭此识别身份

### 2. 数据结构

```c
// 客户端会话（仅活跃用户，内存存储）
typedef struct {
    struct mg_connection *conn;   // 当前连接
    client_state_t state;          // 状态：INIT → CHALLENGED → AUTHED
    int client_id;                 // 用户 ID
    uint8_t public_key[32];        // 公钥（认证阶段临时存储）
    uint8_t challenge[32];         // 挑战数据（认证阶段临时存储）
    char chat_token[64];           // 认证后的 Token
    time_t last_heartbeat;         // 心跳时间
    // ... 业务数据（联系人、文件传输等）
} client_session_t;

// Token 哈希表节点（O(1) 查找）
typedef struct session_node {
    char token[64];
    client_session_t *session;
    struct session_node *next;
} session_node_t;
```

### 3. 关键函数

| 函数 | 说明 |
|------|------|
| `find_session_by_token(token)` | 通过 Token 查找会话 (O(1)) |
| `bind_token_to_session(session, token)` | 绑定 Token 到会话 |
| `generate_challenge()` | 生成 32 字节随机挑战 |
| `generate_chat_token(client_id)` | 生成 64 字符 hex Token |
| `crypto_ed25519_check()` | 验证签名 |

### 4. 安全特性

- ✅ 私钥永不离开客户端
- ✅ 每次登录挑战数据不同（防重放）
- ✅ 挑战数据 5 分钟过期
- ✅ Token 与服务端会话绑定
- ✅ 心跳超时自动清理（120 秒）

### 5. 性能指标

| 指标 | 目标值 |
|------|--------|

| Token 查找 | O(1) |
