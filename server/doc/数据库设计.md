# SQLite 数据库设计

## 设计思路

**核心原则**：数据库存储持久化数据，内存仅存储活跃会话。

```
┌─────────────────────────────────────────┐
│          内存 (快速查找)                 │
│  ┌─────────────────────────────────┐    │
│  │  Token 哈希表                    │    │
│  │  chat_token → session (活跃)    │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
                  ↕ 同步
┌─────────────────────────────────────────┐
│       SQLite (持久化存储)                │
│  ┌─────────────────────────────────┐    │
│  │  clients 表 (用户公钥)           │    │
│  │  challenges 表 (临时挑战)        │    │
│  │  sessions 表 (会话备份)          │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

## 数据库表结构

### clients 表 - 用户公钥

```sql
CREATE TABLE clients (
    client_id INTEGER PRIMARY KEY,      -- 用户 ID
    public_key BLOB NOT NULL,           -- 32 字节 Ed25519 公钥
    created_at INTEGER NOT NULL,        -- 注册时间戳
    last_login INTEGER                  -- 最后登录时间
);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `client_id` | INTEGER | 主键，自增 |
| `public_key` | BLOB | 32 字节公钥 |
| `created_at` | INTEGER | Unix 时间戳 |
| `last_login` | INTEGER | Unix 时间戳 |

### challenges 表 - 临时挑战数据

```sql
CREATE TABLE challenges (
    client_id INTEGER PRIMARY KEY,      -- 用户 ID
    challenge BLOB NOT NULL,            -- 32 字节随机挑战
    created_at INTEGER NOT NULL,        -- 生成时间
    expires_at INTEGER NOT NULL         -- 过期时间（5 分钟）
);

CREATE INDEX idx_challenges_expires ON challenges(expires_at);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `client_id` | INTEGER | 主键，关联 users |
| `challenge` | BLOB | 32 字节随机数 |
| `created_at` | INTEGER | Unix 时间戳 |
| `expires_at` | INTEGER | 过期时间戳 |

### sessions 表 - 会话备份

```sql
CREATE TABLE sessions (
    chat_token TEXT PRIMARY KEY,        -- Token 字符串
    client_id INTEGER NOT NULL,         -- 用户 ID
    conn_ptr INTEGER NOT NULL,          -- 连接指针
    state INTEGER NOT NULL,             -- 会话状态
    last_heartbeat INTEGER NOT NULL     -- 心跳时间
);

CREATE INDEX idx_sessions_heartbeat ON sessions(last_heartbeat);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `chat_token` | TEXT | 64 字符 Token |
| `client_id` | INTEGER | 关联 users |
| `conn_ptr` | INTEGER | mg_connection 指针 |
| `state` | INTEGER | 0=INIT, 1=CHALLENGED, 2=AUTHED |
| `last_heartbeat` | INTEGER | Unix 时间戳 |

## 数据库操作 API

### 初始化

```c
int init_server(const char *db_path);
void cleanup_server(void);
```

### 用户公钥管理

```c
// 保存/更新用户公钥
int db_save_client(int client_id, const uint8_t public_key[32]);

// 加载用户公钥
int db_load_client(int client_id, uint8_t public_key[32]);
```

### 挑战数据管理

```c
// 保存挑战数据
int db_save_challenge(int client_id, const uint8_t challenge[32]);

// 加载挑战数据（验证是否过期）
int db_load_challenge(int client_id, uint8_t challenge[32]);

// 删除挑战数据
int db_delete_challenge(int client_id);

// 清理所有过期挑战
void db_cleanup_expired_challenges(void);
```

### 会话管理

```c
// 保存会话
int db_save_session(const char *token, int client_id, struct mg_connection *conn);

// 删除会话
int db_delete_session(const char *token);

// 清理超时会话
void db_cleanup_timeout_sessions(int timeout);
```

## 并发优化

### WAL 模式

```c
sqlite3_exec(db, "PRAGMA journal_mode=WAL;", NULL, NULL, NULL);
sqlite3_exec(db, "PRAGMA synchronous=NORMAL;", NULL, NULL, NULL);
sqlite3_exec(db, "PRAGMA cache_size=-64000;", NULL, NULL, NULL);  // 64MB 缓存
```

### 锁机制

```c
pthread_mutex_t db_lock;       // SQLite 操作互斥锁
pthread_rwlock_t session_lock; // 会话池读写锁
```

- **读操作**（查找 Token）：共享锁，允许多个并发
- **写操作**（数据库写入）：互斥锁，串行执行

## 内存 vs 数据库

| 数据类型 | 存储位置 | 原因 |
|----------|---------|------|
| 活跃会话 | 内存 + 数据库备份 | 快速查找 + 持久化 |
| 用户公钥 | 数据库 | 节省内存，持久化 |
| 挑战数据 | 数据库 | 临时数据，5 分钟过期 |
| Token 索引 | 内存哈希表 | O(1) 查找性能 |

## 性能指标

| 操作 | 延迟 |
|------|------|
| Token 查找（内存） | < 1 μs |
| 数据库读取 | < 100 μs |
| 数据库写入 | < 1 ms |
| 认证流程 | < 10 ms |
